Alpaca Batch Command Helper

Paste API key, API secret, and one or more Alpaca commands. Each non-empty line in COMMAND is treated as a separate request. A command can be a near-raw Alpaca path, a full Alpaca URL, or a simplified method+path request.

Intended workflow:
1. Your trading assistant tells you exactly what to paste into COMMAND
2. You click Run commands
3. If browser fetch fails for one or more lines, click Copy cURL fallback and run the generated batch in terminal
4. Copy the OUTPUT back into GPT

Reference: Alpaca Getting Started docs

Inputs

Accepted command styles:
• One command per line
• Path only, starting with /
• Full Alpaca URL
• Optional leading method, e.g. GET /v2/stocks/bars?symbols=NVDA&timeframe=5Min&limit=30
• Optional base shortcuts: /v2/stocks/..., /v2/crypto/..., /v2/options/..., /v2/account, /v2/orders
• Lines starting with # are ignored as comments

The helper is intentionally generic. It does not hardcode only one small subset of endpoints. If a line points to a valid Alpaca endpoint and the browser permits the request, the helper will attempt it and return a combined output block containing, for each command:
• the resolved URL
• a compact summary when possible
• the raw JSON

Direct browser fetch may fail because a local HTML file can hit browser security/CORS restrictions. The cURL fallback is included for that reason.
